Overview

The Log Analyzer provides CDN intelligence through a sub-module architecture. It parses, analyzes, and visualizes CDN log data from multiple providers, generates anomaly alerts, and produces DOCX reports with 21 Plotly charts.

Core Capabilities

  • Sub-module architecture: pluggable CDN provider support (Akamai, Medianova, ...)
  • 22-field DataStream 2 parser with auto-detection (TSV/CSV/JSON)
  • Anomaly detection with configurable thresholds

  • DOCX report generation with charts, metrics, and agent summary
  • Date-range fetch from S3 and BigQuery export
  • Settings UI with encrypted credential storage

Sub-Module Architecture

BaseSubModule Pattern

All CDN sub-modules inherit from BaseSubModule and must implement the standard interface:

class BaseSubModule(ABC):
    @abstractmethod
    def parse(self, content: str) -> list[LogEntry]: ...
    @abstractmethod
    def analyze(self, entries: list[LogEntry]) -> Metrics: ...
    @abstractmethod
    def detect_anomalies(self, metrics: Metrics) -> list[Anomaly]: ...
    @abstractmethod
    def generate_charts(self, metrics, entries) -> dict: ...
    @abstractmethod
    def generate_report(self, tenant_id, metrics, anomalies, charts) -> str: ...

SubModuleRegistry

Sub-modules register via REGISTRY dict. The API routes delegate to the appropriate sub-module based on the request.

Critical Metric Calculation Rules

  • Bandwidth is calculated from the bytes field (total transferred), NOT response_body_size
  • Cache Hit Rate uses the cache_hit field (binary 0/1), NOT cache_status
  • Error Rate excludes ERR_CLIENT_ABORT — client aborts are tracked separately as abort_rate
  • Transfer Time analysis applies only to video segment requests (content_type video/* or path *.ts/*.m3u8)

DOCX Report Structure

Report Sections

  1. Executive Summary — Agent-generated bilingual summary
  2. Key Metrics — Total requests, error rate, cache hit rate, TTFB
  3. Charts — All 21 charts as embedded images (kaleido==0.2.1)
  4. Anomalies — Detected anomalies with severity and recommendations
  5. Top Errors — Error code breakdown with counts
  6. Edge & Geo — Geographic and edge server distribution

Reports saved to: data/reports/{tenant_id}/

Parser Auto-Detection

Format Detection Logic

The parse_auto() function detects the log format automatically:

  1. If content starts with { or [ → JSON format
  2. If first line contains tab characters and exactly 22 fields → TSV (DataStream 2 native)
  3. If first line contains commas with known headers → CSV (legacy format)

TSV is the primary format for DataStream 2. CSV mapping is maintained for backward compatibility with older log exports.

Anomaly Detection Thresholds

Thresholds are configurable per tenant via the anomaly_rules SQLite table. The agent respects custom thresholds when they exist and falls back to defaults otherwise.
