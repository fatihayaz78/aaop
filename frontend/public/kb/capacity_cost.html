Forecast Model

# Daily capacity forecast algorithm:
# 1. Load 30-day historical traffic (same day-of-week)
# 2. Apply seasonal adjustment (weekend/weekday, holiday)
# 3. Add live event uplift from scheduled events
# 4. Apply growth trend multiplier
# 5. Generate hourly capacity targets with 20% headroom

for hour in range(24):
    base = historical_avg[dow][hour]
    adjusted = base * seasonal_factor * growth_trend
    if live_events_at(hour):
        adjusted += predicted_viewers * per_viewer_resource
    forecast[hour] = adjusted * 1.20  # 20% headroom

Automation Agent (M04)

Multi-step workflow engine with rollback support, powered by Haiku LLM.

Agent Configuration

LLM ModelHaiku (batch, cost-efficient)
Source Fileapps/capacity_cost/agent.py
Tools Count3 tools (1 LOW, 1 MEDIUM, 1 HIGH)
Workflow StorageSQLite (platform.db)
Multi-Step Workflows: Each automation job consists of ordered steps. Each step has a type (scale, notify, check, wait), target resource, and a rollback action that executes if the step fails.

Workflow Structure

{
  "job_id": "auto_job_2026_03_28_001",
  "trigger": "live_event_starting",
  "steps": [
    {
      "order": 1,
      "type": "scale",
      "target": "cdn_origin_pool",
      "action": "scale_up",
      "params": { "instances": 4 },
      "rollback": { "action": "scale_down", "instances": 4 }
    },
    {
      "order": 2,
      "type": "check",
      "target": "health_endpoint",
      "timeout_seconds": 60
    },
    {
      "order": 3,
      "type": "notify",
      "target": "ops_team",
      "message": "Pre-scale complete for live event"
    }
  ]
}
HIGH Risk Tools: Both create_automation_job and execute_scale_action require admin approval. Automation jobs that modify infrastructure are logged to the audit trail.

▼

Data Model

SQLite for automation jobs and thresholds, DuckDB for capacity analytics.

automation_jobs (SQLite: platform.db)

CREATE TABLE automation_jobs (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    trigger_type    VARCHAR NOT NULL,  -- manual|live_event|threshold|schedule
    trigger_ref     VARCHAR,
    status          VARCHAR DEFAULT 'pending',  -- pending|approved|running|completed|failed|rolled_back
    steps_json      TEXT NOT NULL,  -- JSON array of workflow steps
    created_at      TIMESTAMP NOT NULL,
    approved_at     TIMESTAMP,
    started_at      TIMESTAMP,
    completed_at    TIMESTAMP,
    error_message   TEXT
);

capacity_thresholds (SQLite: platform.db)

CREATE TABLE capacity_thresholds (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    region          VARCHAR NOT NULL,
    metric          VARCHAR NOT NULL,  -- cpu|memory|bandwidth|connections
    warning_value   DOUBLE NOT NULL,
    critical_value  DOUBLE NOT NULL,
    unit            VARCHAR NOT NULL,  -- percent|count|gbps
    updated_at      TIMESTAMP NOT NULL,
    UNIQUE(tenant_id, region, metric)
);

capacity_metrics (DuckDB)

CREATE TABLE capacity_metrics (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    region          VARCHAR NOT NULL,
    metric          VARCHAR NOT NULL,
    value           DOUBLE NOT NULL,
    forecast_value  DOUBLE,
    recorded_at     TIMESTAMP NOT NULL
);
