FAQ ▼
POST to /auth/login with email/password. You receive an access_token (15min) and refresh_token (7d). Include the access token as Authorization: Bearer <token> in all subsequent requests.
All data-access endpoints require the X-Tenant-ID header for multi-tenant isolation. The middleware validates that the authenticated user has access to the requested tenant.
Rate limiting is enforced per tenant via Redis. Default: 100 req/min for GET endpoints, 30 req/min for POST/PUT/DELETE. Configurable per tenant in admin settings.

WebSocket Events

Socket.IO based real-time communication. Manager at backend/websocket/manager.py.

Client Events (Emit)

EMITsubscribeSubscribe to a channel (room join)
EMITunsubscribeUnsubscribe from a channel (room leave)
EMITagent_chatSend message to agent via WebSocket

Server Events (Listen)

ONincident_updateIncident status change broadcast
ONalert_newNew alert fired notification
ONagent_streamStreaming agent response tokens
ONmetric_updateReal-time metric data push

Connection Example

import { io } from "socket.io-client";

const socket = io("http://localhost:8000", {
  auth: { token: "Bearer <JWT>" },
  query: { tenant_id: "tenant_001" }
});

socket.emit("subscribe", { channel: "ops:incidents" });

socket.on("incident_update", (data) => {
  console.log("Incident updated:", data.incident_id, data.status);
});

socket.on("agent_stream", (data) => {
  // Append streaming token to chat UI
  appendToken(data.token, data.session_id);
});
