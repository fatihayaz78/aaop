FAQ ▼
Primary correlation keys: subscriber_id links CRM/Billing/Player, session_id chains Player Events, medianova_req_id links Origin to CDN, device_id spans Player/DRM/Push sources.
Varies by source: JSONL.gz (most), TSV (Medianova CDN), CSV (CRM, Billing), JSON (EPG, New Relic). All parsed into DuckDB logs.duckdb for unified querying.
Fields flagged as PII: client_ip, subscriber_id, email, phone, device_id. The PII scrubber redacts these before LLM processing. Raw data in DuckDB retains PII for operational use.
