Purpose

The DevOps Assistant provides platform engineers with an intelligent conversational interface for platform operations. It translates natural language queries into diagnostic insights, suggests relevant runbooks, and can execute approved operational procedures under strict safety controls.

Key Capabilities

Natural Language Queries
Ask questions about platform health, service status, or recent incidents in plain English. The agent translates into the appropriate diagnostic queries.
Runbook Suggestions
Automatically matches symptoms to relevant runbooks via ChromaDB similarity search and presents step-by-step remediation procedures.
Command Safety Analysis
Every suggested command is classified as safe or dangerous. Safe commands get a green indicator; dangerous ones get a red flag with required approval.
Diagnostic Pipeline
Automated health checks, log correlation, and metric analysis to quickly identify root causes of platform issues.

Target Persona

Platform Engineer — Engineers responsible for maintaining AAOP infrastructure, handling incidents, deploying services, and ensuring platform reliability. They need quick access to operational knowledge and safe command execution.

Agent Loop (4-Step)

1. Plan
→
2. Act
→
3. Observe
→
4. Respond
  • Plan: Parse user intent (diagnostic query, runbook request, command execution). Determine which tools are needed.
  • Act: Execute selected tools. For HIGH risk tools, pause and request human approval before proceeding.
  • Observe: Aggregate tool outputs, cross-reference with platform state. Run safety classification on any suggested commands.
  • Respond: Present findings in structured format: diagnosis, suggested runbook, commands with safety labels, next steps.

Retrieval Strategy
Top-3 results are returned for each query. If the best match scores < 0.75, the agent falls back to general LLM reasoning instead of presenting potentially irrelevant runbook matches.

Tools Reference

8 tools total: 4 LOW 1 MEDIUM 2 HIGH + 1 internal

ToolRiskDescriptionApproval
search_runbooks LOW Semantic search across ChromaDB runbooks collection No
get_service_health LOW Check current health status of platform services via Redis cache No
query_platform_logs LOW Search DuckDB for recent log entries with filters No
get_incident_history LOW Retrieve past incidents from DuckDB for pattern matching No
analyze_command_safety MEDIUM Classify a command as safe or dangerous with reasoning No
execute_runbook HIGH Execute a runbook's command sequence on the target environment Yes
restart_service HIGH Restart a specific platform service with pre/post health checks Yes
classify_intent INTERNAL Internal: parse user message to determine intent category No
HIGH Risk Tools
execute_runbook and restart_service require explicit human approval (approval_required=True). Before execution, the agent displays the full command list, expected impact, and rollback plan. All executions are logged to the immutable audit trail.

Tool Schema Example: search_runbooks

{
  "name": "search_runbooks",
  "risk_level": "LOW",
  "parameters": {
    "query": "string",           // Natural language description of the problem
    "collection": "runbooks",    // ChromaDB collection name
    "top_k": 3,                  // Number of results to return
    "min_similarity": 0.75,      // Minimum cosine similarity threshold
    "tenant_id": "string"
  },
  "returns": {
    "results": [
      {
        "runbook_id": "string",
        "title": "string",
        "summary": "string",
        "steps": ["string"],
        "similarity_score": 0.89
      }
    ]
  }
}

Tool Schema Example: restart_service

{
  "name": "restart_service",
  "risk_level": "HIGH",
  "approval_required": true,
  "parameters": {
    "service_name": "string",    // e.g., "fastapi-backend", "redis", "next-frontend"
    "reason": "string",          // Why the restart is needed
    "pre_check": true,           // Run health check before restart
    "post_check": true,          // Run health check after restart
    "timeout_seconds": 60,       // Max wait for service to come back
    "tenant_id": "string"
  }
}
