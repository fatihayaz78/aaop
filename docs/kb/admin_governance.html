Admin & Governance Overview

Multi-tenant management, compliance scanning, audit logging, and API key governance.

Tenant Agent (M12)

CRUD operations for tenant lifecycle management. Config-driven module enablement (no if/else). API key management with AES-256 encryption. Admin role required.

Haiku LLM 3 HIGH tools

Compliance Agent (M17)

Automated weekly compliance scans. Generates compliance reports covering data retention, access control, PII handling, and security posture.

Sonnet LLM All LOW/MEDIUM tools
Config-Driven Architecture: Module access is controlled by a JSON configuration per tenant. No conditional logic in code — the middleware reads module_config and gates access dynamically. Adding a new module requires only a config update.

No If/Else Pattern: The middleware reads module_config[module_name].enabled at request time. If disabled, it returns 403 Forbidden. No module-specific conditional code exists in the router layer.
HIGH Risk Tools (3):

  • delete_tenant — Soft-deletes tenant, 30-day grace period
  • export_audit_log — Exports full audit trail (may contain sensitive actions)

Agent Configuration

LLM ModelSonnet (compliance analysis requires reasoning)

Tools Count4 tools (2 LOW, 2 MEDIUM)
ScheduleWeekly automated scan (configurable)

Weekly Compliance Scan Checks

Data Retention

Verifies that data older than the retention policy is properly archived or deleted. Checks DuckDB, SQLite, and Redis TTLs.

Access Control

Audits role assignments, detects over-privileged accounts, and verifies admin access patterns against baselines.

PII Handling

Scans for PII leakage in logs, analytics tables, and LLM call history. Verifies pii_scrubber effectiveness.

API Key Expiration

Identifies expired or soon-to-expire API keys. Flags tenants with keys older than rotation policy.

API Endpoints

Router prefix: /admin — 9 REST endpoints.

GET /admin/tenants
▼

List all tenants with pagination and status filtering.

Query Params: status (active|suspended|deleted), page, limit

GET /admin/tenants/{id}
▼

Full tenant details: configuration, masked API key, module access, creation date.

POST /admin/tenants
▼

Create a new tenant. Returns tenant record with full API key (shown once only).

Body: { name, admin_email, plan, module_config }

Response: 201 — Tenant object + one-time full API key

PUT /admin/tenants/{id}/config
▼

Update tenant module configuration. Changes take effect immediately.

Body: { module_config }

POST /admin/tenants/{id}/rotate-key
▼

Rotate API key. Old key is immediately invalidated. New key shown once.

HIGH Risk: approval_required=True. Existing integrations will break until updated.
DELETE /admin/tenants/{id}
▼

Soft-delete tenant. 30-day grace period before permanent deletion.

HIGH Risk: approval_required=True. All tenant data will be deleted after grace period.
GET /admin/audit-log
▼

Query audit log entries with filtering by actor, action type, and date range.

Query Params: tenant_id, actor, action, from, to, page

GET /admin/compliance/latest
▼

Retrieve the latest compliance scan report for a tenant.

Query Params: tenant_id

POST /admin/compliance/scan
▼

Trigger an ad-hoc compliance scan. Returns scan ID for status polling.

Body: { tenant_id, checks[] }

Data Model

SQLite tables for tenant management, audit logging, and compliance.

tenants (SQLite: platform.db)

CREATE TABLE tenants (
    id              VARCHAR PRIMARY KEY,
    name            VARCHAR NOT NULL,
    admin_email     VARCHAR NOT NULL,
    plan            VARCHAR DEFAULT 'standard',  -- basic|standard|premium
    status          VARCHAR DEFAULT 'active',  -- active|suspended|deleted
    module_config   TEXT NOT NULL,  -- JSON module enablement
    api_key_hash    VARCHAR NOT NULL,  -- AES-256-GCM encrypted
    api_key_masked  VARCHAR NOT NULL,  -- aaop_****...****XXXX
    api_key_expires TIMESTAMP,
    created_at      TIMESTAMP NOT NULL,
    updated_at      TIMESTAMP NOT NULL,
    deleted_at      TIMESTAMP
);

audit_log (SQLite: platform.db)

CREATE TABLE audit_log (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    actor           VARCHAR NOT NULL,  -- user or system identifier
    action          VARCHAR NOT NULL,  -- create_tenant|update_config|rotate_key|delete_tenant|...
    resource_type   VARCHAR NOT NULL,
    resource_id     VARCHAR,
    before_state    TEXT,  -- JSON snapshot before change
    after_state     TEXT,  -- JSON snapshot after change
    ip_address      VARCHAR,
    user_agent      VARCHAR,
    created_at      TIMESTAMP NOT NULL
);
Immutable: Audit log entries cannot be updated or deleted. The export_audit_log tool (HIGH risk) creates a read-only export file for compliance purposes.

compliance_reports (SQLite: platform.db)

CREATE TABLE compliance_reports (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    scan_date       DATE NOT NULL,
    overall_score   INTEGER NOT NULL,  -- 0-100
    checks_json     TEXT NOT NULL,  -- JSON array of check results
    recommendations TEXT,  -- JSON array of recommendations
    created_at      TIMESTAMP NOT NULL
);
