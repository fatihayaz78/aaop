Viewer Experience Overview

Real-time QoE monitoring and complaint management for OTT streaming platform.

QoE Agent (M02)

Monitors viewer quality-of-experience metrics in real-time. Calculates composite QoE scores, detects degradations, and emits platform-wide events.

Sonnet LLM 6 LOW tools

Complaint Agent (M09)

NLP-powered complaint categorization with sentiment analysis. Routes complaints and escalates critical issues to OpsCenter.

Sonnet LLM 1 HIGH tool
Architecture: Both agents share the viewer_experience app folder. QoEAgent handles telemetry-driven insights; ComplaintAgent handles user-reported issues. They communicate via the Event Bus.

QoE Score Formula

score = 5.0
      - buffering_ratio * 10
      - startup_delay_penalty
      - error_count * 0.3
      - bitrate_penalty

# Constraints: 0.0 <= score <= 5.0
# startup_delay_penalty = max(0, (startup_ms - 2000) / 1000) * 0.5
# bitrate_penalty = max(0, (target_bitrate - actual_bitrate) / target_bitrate) * 1.0

Score Interpretation

RangeLabelAction
4.5 - 5.0ExcellentNo action needed
3.5 - 4.4GoodMonitor trending
2.5 - 3.4FairInvestigate root cause
1.0 - 2.4PoorEmit qoe_degradation event
0.0 - 0.9CriticalAuto-escalate to OpsCenter

QoE Metrics Tracked

Buffering Ratio

Time spent buffering / total playback time. Weight: x10 penalty in formula.

Startup Delay

Time from play-press to first frame. Penalty kicks in above 2000ms.

Error Count

Player errors per session (DRM, codec, network). Weight: x0.3 per error.

Bitrate Quality

Actual vs target bitrate ratio. Lower bitrate = higher penalty.

Sentiment Analysis

Positive

Viewer satisfied, praising content or feature. Confidence > 0.8 required.

Neutral

Informational report without strong emotion. Default fallback.

Negative

Frustrated viewer. If combined with crash/buffering, escalation priority increases.

HIGH Risk Tool: escalate_complaint requires admin approval. It creates an OpsCenter incident with full complaint context, QoE correlation data, and recommended actions.

API Endpoints

Router prefix: /viewer — 8 REST endpoints + 1 WebSocket.

GET /viewer/qoe/dashboard
▼

Returns QoE dashboard summary with average scores, regional breakdown, and active degradations.

Query Params: tenant_id (required), time_range (1h|6h|24h|7d)

Response: 200 — QoE summary object with metrics array

GET /viewer/qoe/by-region
▼

QoE scores filtered by geographic region with ISP-level detail.

Query Params: tenant_id, region, city, isp

Response: 200 — Array of region QoE objects

GET /viewer/qoe/history
▼

Historical QoE time-series data for trend analysis and charting.

Query Params: tenant_id, start, end, granularity (5m|1h|1d)

Response: 200 — Time-series array with timestamps and scores

GET /viewer/complaints
▼

List complaints with pagination, filtering by category and sentiment.

Query Params: tenant_id, category, sentiment, page, limit

Response: 200 — Paginated complaint list

POST /viewer/complaints
▼

Submit a new complaint. Triggers NLP classification automatically.

Body: { tenant_id, user_id_hash, text, device_type, app_version }

Response: 201 — Created complaint with auto-assigned category and sentiment

GET /viewer/complaints/trends
▼

Complaint volume trends grouped by category over time.

Query Params: tenant_id, days (default: 30)

Response: 200 — Trend data per category

POST /viewer/complaints/{id}/escalate
▼

Escalate a complaint to OpsCenter. Requires admin role. Creates an incident.

Body: { reason, priority, correlation_ids }

Response: 200 — Incident ID and escalation status

HIGH Risk: approval_required=True
GET /viewer/sessions
▼

Query viewer sessions with playback metrics for debugging.

Query Params: tenant_id, user_id_hash, device, date

Response: 200 — Session details with QoE scores per session

WS /viewer/ws/qoe-live
▼

Real-time WebSocket stream of QoE scores and degradation alerts.

Events: qoe_update, degradation_alert, complaint_new

Protocol: Socket.IO via shared/websocket/manager.py

Cross-App Events

Event Bus integration for inter-app communication.

OUTPUT Events (Published)

qoe_degradation

Emitted when average QoE score for a region drops below 2.5 for more than 5 minutes.

{
  "event_type": "qoe_degradation",
  "severity": "HIGH",
  "region": "europe-west1",
  "avg_score": 1.8,
  "affected_viewers": 12450,
  "degradation_factors": ["buffering", "errors"],
  "timestamp": "2026-03-28T14:30:00Z"
}

Consumers: OpsCenter (auto-incident), Alert Center (notification), Capacity & Cost (scale check)

INPUT Events (Consumed)

analysis_complete

From Log Analyzer. When log analysis finishes, QoE Agent correlates findings with viewer experience data to enrich degradation context.

live_event_starting

From Live Intelligence. QoE Agent adjusts thresholds and monitoring frequency for live event periods (increased viewer load).

Data Model

DuckDB tables in analytics.duckdb for QoE and complaint data.

qoe_metrics (DuckDB)

CREATE TABLE qoe_metrics (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    user_id_hash    VARCHAR NOT NULL,
    session_id      VARCHAR NOT NULL,
    region          VARCHAR,
    city            VARCHAR,
    isp             VARCHAR,
    device_type     VARCHAR,
    qoe_score       DOUBLE NOT NULL,
    buffering_ratio DOUBLE,
    startup_ms      INTEGER,
    error_count     INTEGER,
    actual_bitrate  INTEGER,
    target_bitrate  INTEGER,
    recorded_at     TIMESTAMP NOT NULL
);

complaints (DuckDB)

CREATE TABLE complaints (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    user_id_hash    VARCHAR NOT NULL,
    text            TEXT NOT NULL,
    category        VARCHAR,  -- buffering|audio|subtitle|crash|billing|other
    sentiment       VARCHAR,  -- positive|neutral|negative
    confidence      DOUBLE,
    device_type     VARCHAR,
    app_version     VARCHAR,
    status          VARCHAR DEFAULT 'open',
    escalated       BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP NOT NULL
);
PII Policy: Only user_id_hash is stored. Raw user IDs are never persisted. The shared/utils/pii_scrubber.py ensures PII is stripped before any LLM call.
