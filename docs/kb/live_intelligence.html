Live Intelligence Overview

Live event management, external data integration, and pre-scaling orchestration for broadcast operations.

Live Event Agent (M05)

Manages live event lifecycle from scheduling to completion. Emits pre-scale events T-30min before start, monitors concurrent viewers, and coordinates with Capacity Agent.

Sonnet LLM 1 HIGH tool

External Data Agent (M11)

Polls external services (SportRadar, DRM, EPG) at configurable intervals. Enriches platform context with real-time sport scores, DRM health, and schedule data.

Sonnet LLM 1 HIGH tool
T-30 Minute Rule: When a live event is scheduled, at T-30 minutes the system emits a live_event_starting event. If predicted viewers exceed 50K, Capacity Agent automatically triggers pre-scaling to handle the surge.

Viewer Prediction Model

The agent predicts concurrent viewers using historical data combined with event metadata:

# Prediction factors:
# 1. Historical average for same event type (e.g., football derby)
# 2. Day-of-week and time-slot multiplier
# 3. Team popularity index
# 4. Season/tournament weight
# 5. Recent growth trend (last 4 events)

predicted_viewers = base_average * dow_multiplier * popularity * season_weight * trend
# If predicted_viewers > 50,000 → trigger pre-scale
HIGH Risk Tool: trigger_pre_scale initiates infrastructure scaling based on predicted viewer count. Requires approval when predicted viewers exceed 100K.

API Endpoints

Router prefix: /live — 7 REST endpoints + 1 WebSocket.

GET /live/events
▼

List all live events with filtering by status (scheduled, live, completed).

Query Params: tenant_id, status, date_from, date_to

GET /live/events/{id}
▼

Get full event details including viewer prediction, actual viewers, and scaling status.

POST /live/events
▼

Create a new live event with schedule, type, and expected viewers.

Body: { tenant_id, title, event_type, start_time, expected_viewers, sport_match_id }

GET /live/external/sport/{match_id}
▼

Get cached SportRadar data for a specific match. Returns live scores and events.

GET /live/external/drm/health
▼

Current DRM license server health: latency, error rate, active provider.

POST /live/events/{id}/pre-scale
▼

Manually trigger pre-scaling for an event. Requires admin approval.

HIGH Risk: approval_required=True
POST /live/external/drm/fallback
▼

Override DRM to fallback provider for a duration. Requires admin approval.

Body: { duration_minutes, reason }

HIGH Risk: approval_required=True
WS /live/ws/events
▼

Real-time WebSocket stream of live event updates, viewer counts, and external data changes.

Events: viewer_count_update, event_status_change, external_data_refresh

Cross-App Events

Event Bus integration for inter-app communication.

OUTPUT Events (Published)

live_event_starting

Emitted T-30 minutes before a scheduled live event. Triggers pre-scaling in Capacity Agent and threshold adjustments in QoE Agent.

{
  "event_type": "live_event_starting",
  "event_id": "evt_derby_2026_03_28",
  "title": "Galatasaray vs Fenerbahce",
  "predicted_viewers": 185000,
  "start_time": "2026-03-28T20:00:00Z",
  "pre_scale_required": true
}

Consumers: Capacity & Cost (pre-scale), Viewer Experience (threshold adjustment), Alert Center (monitoring mode)

external_data_updated

Emitted when external data sources provide significant updates (score change, DRM issue, schedule change).

{
  "event_type": "external_data_updated",
  "source": "sportRadar",
  "match_id": "sr_match_12345",
  "update_type": "goal_scored",
  "data": { "score": "2-1", "minute": 73 }
}

Consumers: Growth & Retention (engagement correlation), OpsCenter (context enrichment)

INPUT Events (None)

Note: Live Intelligence is primarily an event producer. It does not consume events from other apps — it is the source of truth for live event and external data state.

Data Model

DuckDB analytics + SQLite platform tables for live events and external data.

live_events (SQLite: platform.db)

CREATE TABLE live_events (
    id              VARCHAR PRIMARY KEY,
    tenant_id       VARCHAR NOT NULL,
    title           VARCHAR NOT NULL,
    event_type      VARCHAR NOT NULL,  -- football|basketball|concert|other
    status          VARCHAR DEFAULT 'scheduled',  -- scheduled|pre_scale|live|completed
    start_time      TIMESTAMP NOT NULL,
    end_time        TIMESTAMP,
    predicted_viewers   INTEGER,
    peak_viewers        INTEGER,
    sport_match_id      VARCHAR,
    pre_scale_triggered BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP NOT NULL
);

external_data_cache (DuckDB)

CREATE TABLE external_data_cache (
    id          VARCHAR PRIMARY KEY,
    tenant_id   VARCHAR NOT NULL,
    source      VARCHAR NOT NULL,  -- sportRadar|drm|epg
    source_id   VARCHAR NOT NULL,
    data_json   TEXT NOT NULL,
    fetched_at  TIMESTAMP NOT NULL,
    stale       BOOLEAN DEFAULT FALSE
);
