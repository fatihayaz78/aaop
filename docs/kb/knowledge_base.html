ARCHITECTURE.md

Architecture Decision Records

  • ADR-001: Local-first with GCP migration path (SQLite → Spanner, DuckDB → BigQuery, ChromaDB → Vertex AI VS)
  • ADR-002: LangGraph 4-step agent loop (Perceive → Reason → Act → Reflect)
  • ADR-003: Event Bus via asyncio.Queue (9 event types, fan-out, cross-app wiring)
  • ADR-004: Multi-tenant via X-Tenant-ID header injection middleware
  • ADR-005: LLM Gateway with 3-tier routing (Haiku/Sonnet/Opus) + retry + cache + cost tracking
  • ADR-006: Tool risk labeling (LOW/MEDIUM/HIGH) with approval gate for HIGH
  • ADR-007: DuckDB for analytics, SQLite for metadata, ChromaDB for vector search
  • ADR-008: Redis for cache/TTL, pub/sub for real-time events
  • ADR-009: Next.js 14 with shadcn/ui + Recharts for frontend
  • ADR-010: structlog for all logging, PII scrubbing before LLM calls
  • ADR-011: CI pipeline: ruff + mypy + pytest (coverage ≥ 80%)

Migration Map

shared/clients/ adaptor pattern sayesinde GCP gecisi sadece client dosyalarini degistirerek yapilir: sqlite_client → Spanner, duckdb_client → BigQuery, chroma_client → Vertex AI Vector Search, redis_client → Memorystore.

Event Bus

9 event type: incident_created, incident_updated, rca_completed, alert_fired, alert_resolved, qoe_degradation, cdn_anomaly, live_event_started, live_event_ended. Fan-out: Tek event birden fazla consumer tarafindan dinlenebilir.

Red Lines

  • Agent icerisinde dogrudan DB erisimi YASAK (sadece tools uzerinden)
  • tenant_id olmadan tool cagrisi YASAK
  • print() yerine structlog kullanilmali
  • Python 3.14 YASAK, kaleido 1.x YASAK

CDN Outage Response Runbook

P0 CDN

  1. Alert Teyidi: Alert Center'dan gelen CDN anomaly alert'ini dogrula. Grafana/New Relic dashboard'larini kontrol et.
  2. Etki Analizi: Log Analyzer'dan son 15 dakikanin CDN latency ve error rate metriklerini cek. Etkilenen kullanici sayisini belirle.
  3. Incident Olustur: Ops Center'da P0 incident ac. Tum ilgili taraflar otomatik bilgilendirilir.
  4. CDN Provider Iletisim: Medianova/Akamai NOC'a ticket ac. Escalation: 15dk icinde cevap gelmezse manager'a bildir.
  5. Trafik Yonlendirme: Medianova kesintisinde trafigi Akamai'ye yonlendir (DNS CNAME degisikligi veya load balancer rule).
  6. Monitoring: Trafik yonlendirme sonrasi 5dk boyunca latency ve error rate'i izle. QoE skorlarini kontrol et.
  7. Kullanici Bildirimi: Push notification ve in-app banner ile kullanicilari bilgilendir (kisa kesinti yasandigini, cozuldugunu belirt).
  8. Post-Mortem Baslat: Incident resolved sonrasi 24 saat icinde RCA toplantisi planla. Ops Center'da RCA baslat.
  9. KB Guncelle: RCA tamamlaninca Knowledge Base'e incident raporu ve guncel runbook ekle.

DRM Certificate Renewal Runbook

P1 DRM

  1. Sertifika Kontrolu: FairPlay ve Widevine sertifika bitiS tarihlerini kontrol et. Alert Center'dan 30 gun oncesinden uyari gelir.
  2. Yeni Sertifika Talebi: Apple Developer Portal (FairPlay) veya Google Widevine Portal'dan yeni sertifika talep et. CSR olustur ve gonder.
  3. Test Ortami: Yeni sertifikayi staging ortaminda test et. Player'da DRM korumali icerik oynatmayI dogrula. iOS (FairPlay) ve Android (Widevine) cihazlarda test yap.
  4. Production Deploy: Maintenance window icinde sertifika degisikligini production'a al. Rollback plani hazir olsun.
  5. Dogrulama: Production'da DRM korumali icerik oynatmayi dogrula. Hata orani izle, QoE skorlarini kontrol et. 24 saat monitoring.

P0 Incident Response Runbook

P0 General

  1. Incident Tanimla: Alert'i dogrula, P0 seviyesini onayla. >1000 kullanici etkileniyorsa veya core servis down ise P0.
  2. Incident Commander Ata: On-call muhendis Incident Commander (IC) olur. IC tum koordinasyonu yonetir.
  3. War Room Ac: Slack #incident-war-room kanalini aktive et. Tum ilgili muhendisleri davet et.
  4. Etki Belirle: Etkilenen servisler, kullanici sayisi, gelir etkisi. Log Analyzer ve Viewer Experience dashboard'larinI kullan.
  5. Mitigation Baslat: Hizli cozum uygula (traffic shift, rollback, restart). Kalici cozum sonra yapilir.
  6. Stakeholder Bilgilendirme: Her 15 dakikada status update. CTO/VP-Eng'e dogrudan bildirim.
  7. Cozum Dogrula: Metrikler normale dondu mu? QoE skorlari recovery gosteriyor mu? 15dk stability window bekle.
  8. Incident Kapat: Ops Center'da incident'i resolved yap. Timeline'i tamamla.
  9. RCA Planla: 24-48 saat icinde blameless post-mortem toplantisi. RCA dokkmanini Knowledge Base'e ekle.

Incident Report: CDN Outage (28 Feb 2026)

P0 Resolved

Summary

28 Subat 2026, 14:23 UTC'de Medianova Istanbul POP'ta routing loop tespit edildi. Turkiye trafiginin %100'u etkilendi. Latency 2 saniyeyi asti, origin server'lar 5xx hatasi dondurmeye basladi.

Timeline

  • 14:23 — Alert Center: CDN latency anomaly (auto-detected)
  • 14:25 — Ops Center: P0 incident olusturuldu (inc-2026-0042)
  • 14:28 — IC atandi, war room acildi
  • 14:35 — Medianova NOC'a ticket acildi
  • 14:42 — Karar: Trafik Akamai'ye yonlendirildi
  • 14:55 — Akamai trafik almaya basladi, latency dusuyor
  • 15:10 — QoE skorlari recovery gosteriyor
  • 15:47 — Incident resolved

Impact

Etkilenen kullanici: ~45,000. Toplam downtime: 84 dakika. Estimated revenue impact: ~$12,000.

Root Cause

Medianova Istanbul POP'ta BGP routing konfigurasyonundaki bir degisiklik routing loop olusturdu. Tum trafik loop'a girdi ve timeout'a dustu.

Action Items

  • Medianova ile SLA review toplantisi planla
  • Otomatik CDN failover mekanizmasi gelistir
  • Multi-CDN active-active yapisina gec

Incident Report: FairPlay Cert Expire (15 Mar 2026)

P1 Resolved

Summary

15 Mart 2026, 09:15 UTC'de FairPlay DRM sertifikasinin suresi doldu. iOS cihazlarda DRM korumali icerik oynatimi basarisiz oldu. Android (Widevine) etkilenmedi.

Timeline

  • 09:15 — Alert Center: DRM error rate spike (iOS only)
  • 09:18 — Ops Center: P1 incident olusturuldu
  • 09:22 — Root cause belirlendi: FairPlay cert expired
  • 09:30 — Apple Developer Portal'dan yeni sertifika indirildi
  • 09:45 — Staging'de test edildi, basarili
  • 09:55 — Production'a deploy edildi
  • 10:05 — iOS DRM oynatim normale dondu
  • 10:20 — Incident resolved

Impact

Etkilenen kullanici: ~18,000 (iOS users). Toplam downtime: 65 dakika. Sadece DRM korumali icerik etkilendi, ucretsiz icerikler normal calisti.

Root Cause

FairPlay sertifika yenileme hatirlatici alert'i 30 gun oncesinden tetiklenmisti ancak aksiyona alinmamisti. Cert yenileme sureci manuel oldugu icin unutuldu.

Action Items

  • Cert expiry alert'ini P0 seviyesine yukselt (son 7 gun)
  • Otomatik cert renewal pipeline olustur
  • DRM cert durumunu health check'e ekle
