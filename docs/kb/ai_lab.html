Purpose

The AI Lab is the experimentation and governance hub for all AI/ML workloads in AAOP. It enables AI/ML engineers to run prompt A/B tests, track LLM costs per experiment, manage model lifecycles, and ensure production model quality through drift detection and compliance auditing.

Key Capabilities

Prompt A/B Testing
Test multiple prompt variants with statistical significance tracking (p-value, confidence intervals).
Cost Tracking
Per-experiment LLM token usage and cost monitoring with budget alerts at >80% threshold.
Model Registry
Versioned model snapshots with metadata, performance baselines, and rollback capability.
Drift Detection
Automated model performance drift monitoring using KL-divergence and PSI metrics every 6 hours.

Target Persona

AI/ML Engineer — Data scientists and ML engineers who design, test, and deploy AI models and prompts across the AAOP platform. They need controlled experimentation, cost visibility, and governance guardrails.

Statistical Significance

  • p-value threshold: < 0.05 (configurable per experiment)
  • Confidence interval: 95% CI computed via bootstrap sampling
  • Minimum sample size: 100 responses per variant before significance testing begins
  • Metrics tracked: response quality score, latency (p50/p95/p99), token count, user satisfaction

LLM Cost Tracking

Each experiment has an assigned token budget. The agent tracks input/output token consumption per variant and calculates cost using per-model pricing tables.

Budget Warning at >80%
When token usage exceeds 80% of the experiment budget, a budget_alert event is emitted to the Event Bus. At 95%, the experiment is auto-paused to prevent runaway costs.

Model Registry

The model registry is the single source of truth for all deployed AI models across the platform. Each entry includes:

  • Version ID: Semver-like (e.g., v2.3.1) with immutable snapshot
  • Baseline metrics: Quality score, latency, token efficiency at time of registration
  • Deployment status: STAGING, CANARY, PRODUCTION, RETIRED
  • Audit trail: Who deployed, when, approval chain, rollback history

Compliance & Audit

Every model deployment, configuration change, and rollback is logged in an immutable audit trail stored in DuckDB. The governance agent can generate compliance reports on demand, covering:

  • Model lineage and provenance
  • Data source attestation
  • Approval chain verification
  • Performance SLA adherence
  • Cost compliance per budget allocation

HIGH Risk Tools
update_model_config and switch_model_production require explicit human approval (approval_required=True). The agent will pause execution and wait for confirmation before proceeding. All HIGH risk invocations are logged to the audit trail.
